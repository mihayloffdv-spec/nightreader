# NightReader Spike #1: Dark Mode — Результаты

## Сводка

Оба подхода реализованы, протестированы на 3 PDF-файлах (text-only, text+images, colored diagrams).
Скриншоты всех 9 комбинаций в папке `screenshots/`.

## Таблица результатов

| Критерий | Подход E (Simple) | Подход D (Smart) |
|---|---|---|
| **Текст читаемый?** | Да — белый/кремовый текст на чёрном фоне, отличный контраст | Да — голубовато-белый текст на тёмном фоне, хороший контраст |
| **Картинки приемлемые?** | Терпимо — цвета инвертированы (синий→оранжевый, красный→циановый, зелёный→пурпурный), но различимы | Терпимо — цвета инвертированы аналогично, слегка приглушены, чуть лучше за счёт коррекции яркости/контраста |
| **Text selection работает?** | **Да** — overlay не блокирует жесты (isUserInteractionEnabled = false) | **Под вопросом** — PDFPage subclass проксирует selection-методы к оригинальной странице, но реальное поведение на устройстве требует ручной проверки |
| **Поиск работает?** | **Да** — документ не модифицируется, PDFKit search работает на оригинале | **Нет** — документ заменяется на обёрнутый DarkModePDFPage, поиск по новому документу может не работать корректно |
| **FPS при скролле** | ~60fps — overlay не влияет на рендеринг страниц | ~30-40fps (оценка) — каждая страница рендерится дважды (offscreen + CIFilter), тяжёлая операция |
| **Артефакты при зуме?** | Нет — overlay масштабируется вместе с PDFView | Возможны — offscreen рендер с фиксированным scale=2x при сильном зуме даст размытие |
| **Время рендера страницы** | ~0ms (только compositing) | ~50-200ms (offscreen render + CIFilter chain) |

## Визуальный анализ по PDF-файлам

### 1. Text Only
- **Simple**: Чёрный фон, кремовый текст. Выглядит как настоящий dark mode. Заголовки и тело текста хорошо читаются. Разделители между страницами тёмно-жёлтые.
- **Smart**: Тёмно-синий/чёрный фон, голубовато-белый текст с тёплым оттенком. Тоже читаемый, но менее контрастный чем Simple. Заголовки с синеватым оттенком.

### 2. Text + Images
- **Simple**: Текст инвертирован корректно. Цветные прямоугольники-плейсхолдеры инвертированы (синий→оранжевый, красный→циановый, зелёный→пурпурный). Различимы, но цвета неестественные.
- **Smart**: Аналогичная инверсия картинок, но с дополнительной коррекцией яркости — чуть мягче. Фон между страницами остался белым (проблема: фон вне PDF-страниц не обрабатывается в Smart-режиме).

### 3. Colored Diagrams
- **Simple**: Цветные индикаторы инвертированы, но остаются различимыми. Таблица с тёмными заголовками (инвертирована синяя шапка → оранжевая). Столбчатая диаграмма цветная, но палитра инвертирована.
- **Smart**: Аналогично, но цвета чуть приглушены. Таблица читаемая. Диаграмма различима с приглушённой палитрой.

## Найденные проблемы

### Подход E (Simple)
1. ~~`CALayer.filters` не работает на iOS~~ — **исправлено**: заменён на `compositingFilter` через отдельные UIView overlay
2. Инверсия затрагивает ВСЁ включая UI элементы PDFView (но пикеры вне PDFView — не затрагиваются)
3. Фотографии/изображения инвертируются полностью — неестественные цвета

### Подход D (Smart)
1. `PDFPage.attributedString(for:)` и `character(at:)` не overridable в iOS 26 SDK — убраны из subclass
2. Фон вокруг страниц (background PDFView) не затрагивается — остаётся светлым, создаёт контраст
3. Производительность: двойной рендер каждой страницы (offscreen → CIFilter → draw) — потенциально тяжело на больших PDF

---

## Решение

### MVP: Подход E (Simple) — compositingFilter overlay

**Почему:**
- Мгновенное переключение, нулевая задержка при включении/выключении
- Нет влияния на производительность скролла (~60fps)
- Text selection полностью работает (overlay не перехватывает жесты)
- Поиск работает (документ не модифицируется)
- Простая реализация (~30 строк кода)
- Не ломает зум — overlay масштабируется вместе с view

**Компромиссы для MVP:**
- Картинки инвертируются полностью (неестественные цвета) — приемлемо для ночного чтения текстовых PDF
- Можно добавить настройку "интенсивности" через alpha overlay

### Phase 2: Подход D (Smart) с кэшированием

**Что нужно доработать:**
- Кэширование обработанных страниц (не рендерить каждый раз при скролле)
- Проверить/починить text selection на реальном устройстве
- Исследовать CIKernel (Metal Shading Language) для selective inversion: фон→тёмный, текст→светлый, картинки→слегка затемнить без инверсии
- Решить проблему с фоном PDFView вокруг страниц
- Профилирование через Instruments
