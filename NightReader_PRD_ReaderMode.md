# NightReader — Reader Mode (PRD Addendum)

## Overview

Reader Mode — режим перетекаемого текста (reflowable text), аналогичный чтению ePub в Apple Books. Позволяет менять размер шрифта, при этом текст автоматически переносится по строкам и не уходит за границы экрана.

## Мотивация

PDF — формат с фиксированной разметкой. Текст привязан к координатам на странице, и увеличение шрифта невозможно без перекомпоновки. Reader Mode извлекает контент из PDF и отображает его в нативном SwiftUI ScrollView с настраиваемой типографикой.

## Ключевая идея: гибридный рендеринг

Не весь контент PDF можно корректно отобразить как перетекаемый текст. Стратегия:

1. **Текст** → извлекается, отображается как SwiftUI `Text` с настраиваемым шрифтом
2. **Картинки** (растровые XObject) → извлекаются через CGPDFScanner, вставляются inline
3. **Сложные области** (таблицы, формулы, диаграммы, многоколоночная вёрстка) → рендерятся как растровый снапшот соответствующей области страницы

### Алгоритм анализа страницы

```
Для каждой PDFPage:
1. Извлечь текстовые строки с координатами (PDFSelection API)
2. Оценить качество извлечения текста
   ├── ПЛОХОЕ (сканы, garbled text) → вся страница как снапшот
   └── ХОРОШЕЕ → продолжить анализ
3. Найти "дыры" между текстовыми строками (gap > 30pt)
   └── Каждая дыра → снапшот этой области (картинка/таблица/диаграмма)
4. Собрать контент-блоки, отсортировать сверху вниз
5. Отобразить в ScrollView: Text(...) и Image(snapshot) чередуются
```

### Визуальная структура Reader Mode

```
ScrollView (vertical)
├── Text("Глава 1. Введение")              ← настраиваемый шрифт
├── Text("Длинный абзац текста...")         ← перетекает по ширине экрана
├── Image(snapshot: область с картинкой)     ← aspect fit, ширина экрана
├── Text("Как видно из рисунка...")         ← настраиваемый шрифт
├── Image(snapshot: область с таблицей)      ← растровый снапшот
├── Text("Продолжение текста...")
└── --- page break indicator ---
    ├── Text("Следующая страница...")
    └── ...
```

## Фазы реализации

### Фаза 1: Базовый Reader Mode (MVP) ← ТЕКУЩАЯ
- `PDFContentExtractor` — сервис извлечения контента:
  - Текстовые строки через `PDFSelection.selectionsByLine()` + `bounds(for:)`
  - Gap-based snapshot detection (дыры > 30pt между строками → снапшот)
  - `CGContext.drawPDFPage()` + `clip(to:)` для рендера области в UIImage
  - Оценка качества текста (fallback на полностраничный снапшот)
- `ReaderModeView` — SwiftUI view:
  - ScrollView с Text + Image блоками
  - Слайдер размера шрифта (12–32pt, шаг 1pt)
  - Межстрочный интервал пропорционально размеру шрифта
  - Поддержка тем (цвет фона/текста из текущей темы)
  - Dimmer overlay работает как в PDF-режиме
- Интеграция:
  - Кнопка переключения PDF ↔ Reader в тулбаре
  - Сохранение текущей страницы при переключении
  - Сохранение размера шрифта в UserDefaults

### Фаза 2: Извлечение растровых изображений
- CGPDFScanner: обход контент-стрима, отслеживание CTM
- Извлечение XObject Image с позициями
- Inline-вставка реальных картинок (не снапшотов области)

### Фаза 3: Детекция сложных областей
- Детекция таблиц: кластеры `re`/`m`/`l` операторов (горизонтальные + вертикальные линии)
- Детекция формул: мат-шрифты (CMMI, CMSY, Symbol) + быстрая смена fontSize
- Многоколоночная вёрстка: кластеры левых отступов текстовых строк

### Фаза 4: Полировка
- Анимация перехода между PDF и Reader Mode
- Поиск в Reader Mode
- Закладки в Reader Mode
- Prefetch + кэширование снапшотов (NSCache, LRU, ~5 страниц)

## Технические детали

### Извлечение текста
- API: `PDFPage.selection(for: pageBounds)?.selectionsByLine()`
- Каждая строка: `selection.string` + `selection.bounds(for: page)`
- Сортировка: по Y-координате (maxY descending — сверху вниз в PDF-координатах)

### Рендер области страницы в UIImage
```swift
CGContext(data: nil, width: pixelW, height: pixelH, ...)
ctx.scaleBy(x: scale, y: scale)           // Retina
ctx.translateBy(x: -region.x, y: -region.y)  // сдвиг к области
ctx.clip(to: region)                       // ограничение
ctx.drawPDFPage(page)                      // рисуем
```
- Масштаб: 2x (Retina) — ~7.7 MB на страницу US Letter
- Кэш: NSCache с totalCostLimit ~60 MB

### Оценка качества текста
- Плотность символов (chars / page area)
- Средняя длина слов (2–25 = нормально)
- Порядок чтения (Y-координаты строк убывают)
- Качество < порога → fallback на полностраничный снапшот

### Память
| Масштаб | На страницу | Кэш 5 стр |
|---------|------------|-----------|
| 2x      | ~7.7 MB    | ~38 MB    |
| 3x      | ~17 MB     | ~87 MB    |

Используем 2x + NSCache с автоочисткой при memory warning.

### Потокобезопасность
- `CGPDFDocument` / `CGPDFPage` не потокобезопасны
- Рендер снапшотов: отдельный `DispatchQueue` / `Task.detached`
- Каждый поток использует свой экземпляр `CGPDFDocument`

## Ограничения (известные)

- Таблицы отображаются как снапшоты (не редактируемые)
- Векторные диаграммы попадают в снапшоты через gap-detection
- Сложная многоколоночная вёрстка может нарушить порядок текста → fallback на снапшот
- Сканированные PDF (без текстового слоя) → только полностраничные снапшоты
